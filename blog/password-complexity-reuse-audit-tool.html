Password Complexity & Reuse Audit Tool

A deep dive into auditing cracked credentials for password strength, complexity patterns, and reuse detection across enterprise environments.

Introduction

Weak and reused passwords are a persistent threat, especially after password dumps or internal audits. To help security teams make sense of cracked credential data, I've built the Cracked Password Complexity & Reuse Audit Tool, a utility that inspects cracked passwords for length, complexity, and reuse patterns.

It generates a visual terminal report and an optional CSV export, making it easy to identify the weakest links in an organisation's credential hygiene and present findings to stakeholders.

Why Password Auditing Matters

Even with modern authentication mechanisms, passwords remain the primary gatekeeper for most enterprise systems. Common issues that persist across organisations include:

  • Short passwords that can be brute-forced in seconds.
  • Lack of complexity: passwords missing uppercase, lowercase, digits, or special characters.
  • Password reuse across multiple accounts, amplifying the blast radius of a single compromise.
  • Common patterns such as seasonal passwords (e.g. "Summer2025!") or keyboard walks (e.g. "qwerty123").

After a credential dump or Active Directory audit, security teams often end up with a list of cracked hashes but lack a quick way to assess the overall password health. This tool fills that gap.

Features

The Password Complexity & Reuse Audit Tool provides the following capabilities:

  • Length Analysis: Categorises passwords by length and flags those below minimum thresholds.
  • Complexity Scoring: Evaluates each password against complexity criteria including uppercase, lowercase, digits, and special characters.
  • Reuse Detection: Identifies passwords shared across multiple accounts and highlights the most reused credentials.
  • Visual Terminal Report: Outputs a colour-coded summary directly in the terminal for quick assessment.
  • CSV Export: Optionally exports the full audit results to CSV for further analysis or reporting.
  • Batch Processing: Handles large credential files efficiently, suitable for enterprise-scale audits.

How It Works

The tool accepts a text file containing cracked credentials in the format username:password (one per line), which is the standard output from tools like Hashcat or John the Ripper.

1. Input Parsing

The tool reads the input file and separates usernames from passwords. It handles various formats and edge cases gracefully, including entries with colons in passwords.

2. Complexity Analysis

Each password is evaluated against a set of complexity rules:

  • Minimum length check (configurable, default 8 characters)
  • Presence of uppercase letters (A–Z)
  • Presence of lowercase letters (a–z)
  • Presence of digits (0–9)
  • Presence of special characters (!@#$%^&* etc.)

3. Reuse Detection

The tool groups accounts by their password value and flags any password used by more than one account. The most reused passwords are highlighted in the report.

4. Report Generation

A comprehensive report is generated showing overall statistics, a breakdown of complexity failures, the top reused passwords, and per-account details.

Use Cases

🔐 Post-Penetration Test Reporting

After cracking Active Directory hashes during an engagement, use this tool to quantify and visualise password weaknesses for the client report.

📋 Internal Security Audits

IT security teams can periodically audit their organisation's password health and track improvement over time.

📊 Compliance Evidence

Generate evidence for compliance frameworks like ISO 27001, NIST 800-63, and PCI DSS that require demonstrated password policy enforcement.
